JOB 1 - INPUT DE TAREFAS VIA REUNIAO:

Passo 1.1 - Buscar paginas no INBOX REUNIOES:
Execute via Bash:
curl -s "https://api.notion.com/v1/blocks/331f915e8d4f81278116efcb030aec6e/children?page_size=50" \
  -H "Authorization: Bearer $NOTION_TOKEN" \
  -H "Notion-Version: 2022-06-28"
Use Python para parsear o JSON. Filtre blocos com type=child_page. Anote o id e title de cada um.

Passo 1.2 - Verificar comentarios em cada child_page:
curl -s "https://api.notion.com/v1/comments?block_id=PAGE_ID" \
  -H "Authorization: Bearer $NOTION_TOKEN" \
  -H "Notion-Version: 2022-06-28"
Se algum comentario contiver "Tarefas criadas": pule essa pagina.
Se nenhuma pagina a processar: log "JOB 1: sem reunioes novas" e va para JOB 2.

Passo 1.3 - Ler conteudo de cada pagina a processar:
curl -s "https://api.notion.com/v1/blocks/PAGE_ID/children?page_size=100" \
  -H "Authorization: Bearer $NOTION_TOKEN" \
  -H "Notion-Version: 2022-06-28"
Se has_more=true, pagine com start_cursor ate obter todos os blocos.
Concatene todos os textos. Identifique a data da reuniao. Mapeie owners pelos participantes.
Extraia action items: nome da tarefa, owner, prazo (data_reuniao+7 dias se nao mencionado), contexto.
Ignore tarefas ja concluidas ou apenas revisadas sem nova acao.

Passo 1.4 - Verificar duplicatas para cada tarefa extraida:
curl -s -X POST "https://api.notion.com/v1/databases/31ff915e8d4f8011986bd55487330ecd/query" \
  -H "Authorization: Bearer $NOTION_TOKEN" \
  -H "Notion-Version: 2022-06-28" \
  -H "Content-Type: application/json" \
  -d '{"filter":{"property":"Tarefa","title":{"contains":"TITULO_DA_TAREFA"}}}'
Se encontrar tarefa com owner similar e status diferente de Concluida: e duplicata, pule.

Passo 1.5 - Criar tarefa no Notion para cada tarefa nova:
curl -s -X POST "https://api.notion.com/v1/pages" \
  -H "Authorization: Bearer $NOTION_TOKEN" \
  -H "Notion-Version: 2022-06-28" \
  -H "Content-Type: application/json" \
  -d '{
    "parent": {"database_id": "31ff915e8d4f8011986bd55487330ecd"},
    "properties": {
      "Tarefa": {"title": [{"text": {"content": "NOME_DA_TAREFA"}}]},
      "Owner": {"multi_select": [{"name": "OWNER"}]},
      "Status": {"select": {"name": "Nao iniciada"}},
      "Data limite de entrega": {"date": {"start": "YYYY-MM-DD"}},
      "Conferido": {"checkbox": false}
    },
    "children": [
      {"object":"block","type":"paragraph","paragraph":{"rich_text":[{"text":{"content":"RESUMO_DO_QUE_FAZER"}}]}},
      {"object":"block","type":"heading_2","heading_2":{"rich_text":[{"text":{"content":"Pontos de atencao"}}]}},
      {"object":"block","type":"bulleted_list_item","bulleted_list_item":{"rich_text":[{"text":{"content":"PONTO_1"}}]}},
      {"object":"block","type":"paragraph","paragraph":{"rich_text":[{"text":{"content":"Originado da reuniao: TITULO_PAGINA - DATA_REUNIAO"}}]}}
    ]
  }'
Adicione quantos bulleted_list_item forem necessarios para os pontos de atencao.
Salve a url da pagina criada (campo "url" na resposta).

Passo 1.6 - Adicionar comentario na pagina do INBOX:
curl -s -X POST "https://api.notion.com/v1/comments" \
  -H "Authorization: Bearer $NOTION_TOKEN" \
  -H "Notion-Version: 2022-06-28" \
  -H "Content-Type: application/json" \
  -d '{"parent":{"page_id":"INBOX_PAGE_ID"},"rich_text":[{"text":{"content":"Tarefas criadas em DD/MM/AAAA - N tarefa(s) adicionada(s) ao Controle de Tarefas"}}]}'

Passo 1.7 - Notificar Slack para cada tarefa criada:
curl -s -X POST "$SLACK_WEBHOOK" \
  -H "Content-Type: application/json" \
  -d '{
    "blocks": [
      {"type":"header","text":{"type":"plain_text","text":"NOVA TAREFA ATRIBUIDA"}},
      {"type":"section","text":{"type":"mrkdwn","text":"SLACK_MENTION\n*NOME_TAREFA*\n- Owner: OWNER\n- Prazo: DD/MM/AAAA\n- Status: Nao iniciada\n_CONTEXTO_DA_TAREFA_"}},
      {"type":"section","text":{"type":"mrkdwn","text":"<URL_NOTION|Ver tarefa no Notion>"}}

JOB 2 - PROCESSAMENTO DE EXAMES LABORATORIAIS:

Passo 2.1 - Buscar formularios de anamnese com exames:
curl -s -X POST "https://api.notion.com/v1/databases/329f915e8d4f8040b4cf000be702925d/query" \
  -H "Authorization: Bearer $NOTION_TOKEN" \
  -H "Notion-Version: 2022-06-28" \
  -H "Content-Type: application/json" \

Se nenhum pendente: log "JOB 2: sem exames pendentes" e encerre.

Passo 2.2 - Ler PDF e classificar exames:
GET https://api.notion.com/v1/blocks/FORM_PAGE_ID/children para obter URL do arquivo PDF.
Use WebFetch para ler o PDF.
NOME COMPLETO = titulo do card CRM. Genero M/F pelo primeiro nome.
Extraia exames com valores. Classifique (ADA 2024, Endocrine Society, AHA/ACC, SBC, WHO, KDIGO, ATA, AASLD, Tietz):
Verde=normal, Amarelo=limitrofe, Vermelho=alterado.

Passo 2.3 - Resumo analitico:
Alterados: analise nutricao e performance considerando perfil do cliente.
Normais: linha resumida.

Passo 2.4 - Criar subpagina no card do CRM:
POST https://api.notion.com/v1/pages com parent.page_id=CARD_CRM_ID.
Titulo: Exames Laboratoriais - NOME COMPLETO - DD/MM/AAAA
9 categorias obrigatorias (mesmo sem exames, coloque o header):
HEMOGRAMA, GLICEMIA E METABOLICO, VITAMINAS E MINERAIS, METABOLISMO DO FERRO,
HORMONIOS SEXUAIS, LIPIDIOS, HEPATICO, RENAL, TIREOIDE.
Por categoria: divider + heading_2 caixa alta + tabela (Exame, Referencia, Data, Classificacao).

Passo 2.5 - Comentar no card do CRM:
POST https://api.notion.com/v1/comments com parent.page_id=CARD_CRM_ID e texto com link da subpagina criada.
